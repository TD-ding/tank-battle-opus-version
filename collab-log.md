## 第 1 轮：初始版本（feat）

**目标**：从零搭建可玩的坦克大战核心。

**产出**：
- HTML5 Canvas 单文件实现：13×13 网格、32px 瓦片
- 核心实体：`Game` / `Tank` / `Bullet` 三个类
- 砖墙 / 钢墙 / 草丛 / 基地四种地形与碰撞检测（AABB + 网格瓦片）
- 玩家移动 / 开火、敌人 AI（寻路 + 随机转向 + 间歇开火）
- 关卡推进、计分、生命、最高分（localStorage）持久化
- 帧率无关的增量时间运动

**提交**：`round1: 初始版本` → `round1: review fixes`

---

## 第 2 轮：代码质量优化（refactor）

**自审反馈（模糊化）**：
> “代码里好多写死的数字，速度、间隔什么的散落各处，以后想调参很麻烦；玩家好像能无限连发，手感有点怪。”

**改动**：
- 引入集中配置对象 `CONFIG`，消除散落的魔法数字（速度、子弹速度、开火间隔、刷新间隔、得分、dt 上限等）
- `Tank` 构造函数改为从 `CONFIG` 读取参数
- 新增玩家**同屏子弹上限**（经典坦克大战手感）
- 敌人速度随关卡线性提升的公式参数化
- 单帧 `dt` 上限处理（切后台防穿墙）

**提交**：`round2: refactor - 集中配置 CONFIG...` → 合入 #2

---

## 第 3 轮：用户体验优化（feat / UX）

**自审反馈（模糊化）**：
> “玩起来感觉有点单调，能不能加点打击反馈？另外基地好像太容易被偷家了。”

**改动**：
- 新增 **WebAudio 合成音效系统**（无外部资源）：开火、击中砖/钢墙、爆炸、过关
- 新增**音效开关**（`M` 键），HUD 实时显示音效状态
- **强化基地护墙**：底部两行增加砖墙包围，降低被偷家概率

**自审修复**：
- 兼容浏览器自动播放策略：`AudioContext` 处于 `suspended` 时在用户手势中自动 `resume`

**提交**：`round3: feat - 用户体验优化...` → `round3: review fixes...` → 合入 #3

---

## 第 4 轮：功能增强（feat / Features）

**自审反馈（模糊化）**：
> “现在打法太单一了，能不能加点经典的道具？比如打死敌人掉个宝物什么的，让游戏更有策略性。”

**改动**：
- 新增**道具系统**，三种道具：
  - ★ 星星：火力升级（Lv1→Lv2 射速/弹速提升，Lv3 并排三连发）+ 额外加分
  - ⛊ 护盾：一段时间内免疫敌方子弹
  - ✸ 清屏炸弹：立即摧毁场上所有敌人并计分
- 携带道具的敌人**闪烁高亮**，提示玩家优先击杀
- 道具有存活时间，即将消失时闪烁
- HUD 新增火力等级与护盾状态显示

**自审修复**：
- 道具掉落在钢墙/基地等实心格上时玩家无法进入拾取 → 新增 `findFreeCell` 螺旋寻空，就近重定位

**提交**：`round4: feat - 功能增强...` → `round4: review fixes...` → 合入 #4

## 第 5 轮：Bug 修复（fix）

**自审反馈（模糊化）**：
> “玩的时候发现暂停再回来，坦克有时候会自己往一个方向走；还有打死最后一个敌人时如果自己也被打中了，会判定失败而不是过关，感觉不太对。”

**改动**：
- 修复「卡键」Bug：暂停/恢复、覆盖层按钮回调统一清空按键状态（`clearInput`），避免恢复后坦克自走
- 修复胜负判定优先级：最后一帧与敌人同归于尽时，**优先判定过关**而非失败
- 增强健壮性：切换标签页/最小化（`visibilitychange`）时自动暂停

**提交**：`round5: fix - Bug修复...` → `round5: review fixes...` → 合入 #5

## 验证方式

每轮均通过 Playwright 启动本地静态服务器（`python3 -m http.server`）后：

- 检查浏览器控制台无运行时错误（仅 favicon 404 属预期无害）
- 程序化派发键盘事件（方向键、空格）模拟操作
- 直接单元验证道具逻辑：火力升级封顶、护盾免伤、清屏、道具拾取与过期、落点重定位等

## 工程说明

- 单文件 HTML 游戏，按 `collab-game-dev` 约定**跳过 Docker / CI**。
- 文档：`docs/deployment.md`（运行方式）+ `README.md`。
- 分支策略：每轮从最新 `main` 切 `agent/round<N>`，两次提交后推送并经 PR 合入，合并即删分支。
